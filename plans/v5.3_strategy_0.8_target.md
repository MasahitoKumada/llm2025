## v5.3 戦略プラン

### アプローチ1: 直接改善（短期）

#### 1. ターゲット訓練データの追加（~10件）

エラーパターンに対応する正しいTOML構文の例を追加：

**TOML正しい構文例（inline tableは1行）:**
```toml
# 正しい例（1行inline table）
species = [{ name = "Xylosaur", traits = [{ trait_name = "bioluminescence", trait_value = 0.8 }] }]

# 正しい例（標準テーブル形式）
[[species]]
name = "Xylosaur"
[[species.traits]]
trait_name = "bioluminescence"
trait_value = 0.8
```

**TOML配列テーブル構文:**
```toml
# 間違い
[artifact.inscriptions.list]
language = "Latin"
[artifact.inscriptions.list]  # 重複エラー
language = "Sumerian"

# 正しい（配列テーブル）
[[artifact.inscriptions.list]]
language = "Latin"
[[artifact.inscriptions.list]]
language = "Sumerian"
```

**XML特殊文字エスケープ:**
```xml
<!-- 間違い -->
<title>Steam & Shadows</title>

<!-- 正しい -->
<title>Steam &amp; Shadows</title>
```

### アプローチ2: Sequential Learning（中期）

Person Uの方法を参考に：

```
Stage 1: Base Model + JSON/CSV データ → Model A
```
