# ノートブック バージョニング方針

## 概要

本プロジェクトでは、ノートブック（`.ipynb`）の修正・改善を行う際に、**バージョン管理されたコピーを作成してから編集する**方針を採用する。これにより、過去のバージョンへのロールバックや、バージョン間の差分比較が容易になる。

---

## 命名規則

```
{ベースファイル名}_v{N}.ipynb
```

- **ベースファイル名**: 標準コードのファイル名（拡張子を除く）
- **N**: バージョン番号（1, 2, 3, ...）

### 例

| ファイル | 説明 |
|---|---|
| `メインコンペ_標準コード1(SFT)_v0.ipynb` | 標準コード（オリジナル）。変更しない |
| `メインコンペ_標準コード1(SFT)_v1.ipynb` | v1: ハイパーパラメータ改善版 |
| `メインコンペ_標準コード1(SFT)_v2.ipynb` | v2: 次の改善版（v1をコピーして作成） |

---

## 修正手順

### 1. 現在の最新バージョンを確認する

最新のノートブックを特定する。例: `*_v1.ipynb` が最新であれば、次のバージョンは `v2`。

### 2. コピーを作成する

最新バージョンのノートブックをコピーして、バージョン番号を1つ増やす。

```bash
cp "メインコンペ_標準コード1(SFT)_v1.ipynb" \
   "メインコンペ_標準コード1(SFT)_v2.ipynb"
```

### 3. 新バージョンに対して修正を行う

- **コピー先のファイル（`*_v{N+1}.ipynb`）のみを編集する**
- 元のバージョン（`*_v{N}.ipynb`）は変更しない
- ノートブック先頭のタイトルセル（セル0）に、バージョン番号と変更内容を記載する

### 4. 変更点を記録する

ノートブックのセル0（タイトルマークダウン）に、以下を含める:

- バージョン番号（例: `v2`）
- 変更の概要
- 前バージョンからの差分テーブル（パラメータ変更がある場合）

---

## ルール

### 必須

1. **標準コード（オリジナル）は絶対に変更しない**
   - `メインコンペ_標準コード1(SFT).ipynb` 等のオリジナルファイルは参照専用
2. **既存のバージョンファイルも変更しない**
   - `*_v1.ipynb` が完成したら、以降は `*_v2.ipynb` を作って編集する
3. **最新のバージョンからコピーする**
   - v3 を作る場合は v2 からコピーする（v1 からではない）
4. **バージョン番号は連番にする**
   - v1 → v2 → v3 ...（飛ばさない）

### 推奨

- 各バージョンの変更理由・根拠を `plans/` 配下に記録する
- 大きな方針変更がある場合は、先に `plans/` に計画を書いてからノートブックを修正する
- ノートブック内のコメントで、変更箇所に `★` マークを付けて視認性を高める
